# Beitragen zu morpheus-link-bot

Vielen Dank, dass du Interesse hast, zum **morpheus-link-bot** beizutragen! ❤️  
Jede Form der Unterstützung — ob Bug-Report, Feature-Vorschlag oder Code — ist willkommen.

## 📋 Wie du beitragen kannst

### 1. Bug melden
- Verwende bitte das [Bug Report Template](.github/ISSUE_TEMPLATE/bug_report.md).
- Gib so viele Details wie möglich an (Logs, betroffene Nachricht, Konfiguration, Maubot-Version etc.).
- **Titel** sollte mit `[BUG]` beginnen.

### 2. Feature oder Verbesserung vorschlagen
- Verwende das [Feature Request Template](.github/ISSUE_TEMPLATE/feature_request.md).
- Beschreibe klar den Nutzen und idealerweise einen Anwendungsfall.
- **Titel** sollte mit `[FEATURE]` beginnen.

### 3. Code beitragen (Pull Requests)

1. **Fork** das Repository.
2. Erstelle einen neuen Branch (`git checkout -b feature/meine-tolle-idee`).
3. Mach deine Änderungen.
4. Teste sie lokal (siehe [README.md → Paketerstellung und Deployment](https://github.com/M2tecDev/morpheus-link-bot#paketerstellung-und-deployment)).
5. Erstelle einen **Pull Request** gegen den `main`-Branch.
6. Beschreibe im PR kurz, was geändert wurde und warum.

**Wichtige Hinweise:**
- Der Bot ist ein Maubot-Plugin → halte dich an die Maubot-Konventionen.
- Die gesamte Dokumentation und das User-Interface sind auf **Deutsch**. Neue Strings und Kommentare bitte ebenfalls auf Deutsch halten.
- Füge bei neuen Features oder größeren Änderungen einen Eintrag im [Änderungsprotokoll](https://github.com/M2tecDev/morpheus-link-bot#änderungsprotokoll) hinzu.
- Achte auf die bestehende Code-Qualität (Type Hints, Logging, Error-Handling).

### 4. Andere Beiträge
- Dokumentation verbessern
- Übersetzungen (aktuell nur Deutsch)
- Ideen für neue Wildcard-Features, Performance-Optimierungen, etc.

## Fragen?

Du kannst gerne eine [Issue](https://github.com/M2tecDev/morpheus-link-bot/issues) öffnen oder mir direkt schreiben.

Vielen Dank für deine Unterstützung — zusammen machen wir den Bot noch besser! 🚀
